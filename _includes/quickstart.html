Quickstart

Get up and running with Chronix in 3 minutes with our quickstart guide:.

Prerequisites

You need a Java 8 installed on your system. That's it :-)

Download and Run

The Chronix showcase consists of two parts. The Chronix Server (download) and an example JavaFX application (download) for time series exploration (pick the latest release, respectively). Unzip the Chronix Server (it contains one week of operational time series data), the JavaFX part is an executable Java archive.
Just use the following instructions:

chronix@chronixDB:~$ mkdir chronixShowcase
chronix@chronixDB:~$ cd chronixShowcase
chronix@chronixDB:~$ wget https://github.com/ChronixDB/chronix.server/releases/download/v0.0.2/chronix-0.0.2.zip
chronix@chronixDB:~$ unzip chronix-0.0.2.zip
chronix@chronixDB:~$ cd chronix-solr-5.3.1/
chronix@chronixDB:~$ chmod +x bin/solr
chronix@chronixDB:~$ export JAVA_HOME=/usr/lib/jvm/java-8-oracle/
chronix@chronixDB:~$ /bin/solr start
Started Solr server on port 8983 (pid=3591). Happy searching!
chronix@chronixDB:~$ ▋

Chronix is now running on localhost:8983. Now we can start the JavaFX application. Download the executable jar from GitHub and start it from the console:

chronix@chronixDB:~$ cd chronixShowcase
chronix@chronixDB:~$ wget https://github.com/ChronixDB/chronix.examples/releases/download/v0.0.2/chronix-timeseries-exploration.jar
chronix@chronixDB:~$ chmod +x chronix-timeseries-exploration.jar
chronix@chronixDB:~$ java -jar chronix-timeseries-exploration.jar
Setting up Chronix with a remote solr to URL http://localhost:8983/solr/chronix/
Checking connection to solr. Result true.
chronix@chronixDB:~$ ▋

With the running JavaFX application, we can query Chronix for time series data. A simple query that delivers all time series whose metric name contains Load is metric:*Load*. Just enter the query term into the text box at the top and hit Shift + Enter to retrieve the result. Compute the maximum of the result we add a filter query ag=max in the second text box and press again Shift + Enter. To check if the average load (metric:*Load*avg) has a positive trend one can use the analysis=trend function in the filter query text box. Starting the analysis by pressing Shift + Enter. These queries described and shown in the screencast represent only a few simple queries, check out our examples repository for more details.
